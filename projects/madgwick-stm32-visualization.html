A personal robotics and embedded sensing project developed to test real-time orientation estimation on hardware. Project Overview

This project was created to explore sensor fusion for robotics applications using real hardware. The goal was to implement the Madgwick filter on an STM32F401 microcontroller, process raw IMU data from an MPU9250 sensor, calculate orientation angles, and visualize the motion on a PC in real time.

Sensor Fusion

The system combines accelerometer, gyroscope, and magnetometer data from the MPU9250 to estimate orientation using the Madgwick filter.

Embedded Processing

Orientation estimation is performed directly on the STM32F401, allowing the microcontroller to output processed roll, pitch, and yaw values instead of only raw sensor data.

PC Visualization

A Processing-based visualization receives orientation data through the USB virtual COM port and animates a rectangular body to represent the sensor board motion.

System Architecture

The system is divided into two parts: embedded firmware on the STM32F401 and a PC-side visualization program. The STM32 reads raw motion data, computes orientation using the Madgwick filter, and sends the processed angles to the PC. The PC program converts the data into the correct frame of reference and visualizes the motion.

Professional Project Description

This personal project involved implementing the Madgwick orientation filter on an STM32F401 microcontroller using an MPU9250 inertial measurement unit. The firmware was developed in STM32CubeIDE and reads accelerometer, gyroscope, and magnetometer data from the sensor. The sensor data is processed on the microcontroller to estimate roll, pitch, and yaw angles, which are then transmitted to a host PC through a USB virtual COM port.

A PC-side visualization program was developed using the Processing framework. The program receives orientation data from the COM port, applies a rotation matrix to convert the sensor readings into the PC frame of reference, and maps the resulting orientation to a 3D rectangular body representing the sensor mounted on a breadboard. As the physical breadboard is moved by hand, the virtual body moves accordingly on the computer, providing a real-time visual demonstration of sensor fusion and embedded orientation estimation.
